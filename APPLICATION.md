## 04 项目描述

**项目名称**：PaperCast · 论文电台

**一句话定位**：让最新的 AI 论文，变成 8 分钟可以听完的双人对话播客。

**项目背景与动机**

中文学术圈追前沿论文有一个普遍的痛点：英文 PDF 阅读门槛高、时间成本大，而国内开发者每天的碎片时间（通勤、跑步、做饭）其实非常适合"听"。海外的 NotebookLM 验证了「论文 → 双人播客」这个产品形态的需求，但它目前只对英文母语用户友好，且没有针对学术内容做深度优化。

PaperCast 想做的事情很具体：用户粘贴一个 arXiv 链接，几分钟后拿到一集**完整的中文双人播客**，包含对话脚本、封面图、双人合成音频、带时间戳的文字稿。

**为什么需要 MiMo V2.5**

PaperCast 的每一集播客都会**完整调用 MiMo V2.5 的三种核心能力**，这是我们选择 MiMo 的根本原因 —— 它是国内少数同时具备"长文本推理 + 多模态生成 + 高保真 TTS"的统一平台：

1. **推理（mimo-v2.5-reason）** —— 解析论文 + 生成双人对话脚本（含人设、节奏、专业术语口语化）
2. **多模态（mimo-v2.5-vision）** —— 为每集生成统一视觉风格的封面图
3. **语音合成（mimo-v2.5-tts）** —— 双音色合成，主播 Lin（女声温暖）+ 嘉宾 Wei（男声自然）

**核心场景**

- **场景 A：编辑部周更** —— 每周 5-7 集精选，覆盖 LLM / 视觉 / RL / 系统四大方向
- **场景 B：用户自助生成** —— 登录后粘贴任意 arXiv 链接，按个人节奏生成专属播客
- **场景 C：学习路径** —— 根据兴趣自动编排"零基础到前沿"的播客系列（路线图 Phase 4）

**Token 消耗预估**

| 阶段 | 用量 |
| --- | --- |
| 单集播客 | ≈ 11,000 文本 token + 1 张封面图 |
| 编辑部周更（6 月） | 156 集 ≈ 170 万 tokens |
| 用户自助（6 月，保守估计） | 30,000 集 ≈ 3.3 亿 tokens |
| **合计目标** | **6 个月内 ≈ 3.3 亿 tokens** |

详细的 prompt 设计、调用流程、缓存策略见 [ARCHITECTURE.md](./ARCHITECTURE.md)。

**技术栈**

- Next.js 14（App Router）+ TypeScript：前后端一体
- Tailwind CSS：视觉系统
- MiMo V2.5 Open Platform：所有 AI 能力的唯一来源

**当前进度**

- 端到端 MVP 已完成（mock 模式可完整体验流水线）
- 3 集示例播客（DeepSeek-V3 / Mamba / MiMo V2.5 自家技术解读）已上线
- 真实 MiMo API 接入代码已写完，**等待 Orbit 权益到账即可切换到生产模式**

**为什么我们值得这份权益**

我们是**真的在用 MiMo 创造内容**，不是用来跑通用任务。每一集播客都是 MiMo 三种能力的完整呈现，每一个 token 都最终变成可被搜索、收听、传播的公开内容。这正符合 Orbit 计划"把资源交到真正用 AI 创造的人手中"的目标。
