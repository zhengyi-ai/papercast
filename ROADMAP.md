# Roadmap

PaperCast 的发展分三个阶段，每个阶段都对应一个明确的产品价值与 token 消耗曲线。

---

## Phase 1 · MVP（已完成 ✅）

**目标**：跑通端到端流水线，证明"arXiv URL → 播客"是可行的。

- [x] Next.js 14 全栈骨架
- [x] 三页核心 UI（首页 / 详情页 / 生成页）
- [x] `lib/mimo.ts` 封装 Reason / Vision / TTS 三种调用
- [x] mock 模式：无 key 也能完整体验流水线
- [x] 3 集预制示例（DeepSeek-V3 / Mamba / MiMo V2.5）
- [x] 项目文档与申请素材

---

## Phase 2 · 接入真实 MiMo API（待 Orbit 权益到账）

**目标**：把示例数据切换为 MiMo V2.5 实时生成的真实内容，开始周更。

- [ ] 配置 `MIMO_API_KEY`，关闭 `MIMO_USE_MOCK`
- [ ] 跑通第一集真实播客（用 MiMo V2.5 自家技术报告作为首发）
- [ ] 调优脚本 prompt：人设语气、知识密度、节奏
- [ ] 调优封面 prompt：风格统一性
- [ ] 调优 TTS：句尾停顿、专有名词发音、情感曲线
- [ ] 引入 SSE 流式生成，替代当前的轮询占位
- [ ] 缓存层：相同论文 + 相同风格不重复消耗 token

**预期 token 消耗**：调优阶段约 **500 万 tokens**（包括重生成与对比）。

---

## Phase 3 · 内容运营 + 用户自助

**目标**：从一个 demo 进化成一个有内容、有用户的社区。

- [ ] **编辑部周更**：每周 5-7 集，覆盖 LLM / 视觉 / RL / 系统四大方向
- [ ] **用户自助生成**：登录后可粘贴任意 arXiv 链接生成专属播客
- [ ] **PDF 直传**：扩展到 NeurIPS / ICML / ICLR 等会议论文
- [ ] **Newsletter 同步**：每周精选邮件推送
- [ ] **多语言**：英文版 host / guest 配音，覆盖海外学术圈
- [ ] **可订阅 RSS**：直接接入苹果播客、小宇宙

**预期 token 消耗**：6 个月内累计 **3 亿 tokens** 以上（详见 [ARCHITECTURE.md §4](./ARCHITECTURE.md)）。

---

## Phase 4 · 长期愿景

- [ ] **跨论文综述模式**：选 5 篇相关论文，AI 主播圆桌讨论
- [ ] **作者邀请模式**：把论文一作的真实演讲音频接入，与 AI 主播混合
- [ ] **学习路径模式**：根据用户兴趣，自动编排"零基础到前沿"的播客系列
- [ ] **MiMo 自主学习反馈**：用户对每集的"听完率""反复播放点"反向优化 prompt
